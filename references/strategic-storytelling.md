# Strategic Storytelling — Phase 1 Methodology

Build compelling storylines for C-level presentations by combining the empathy and iteration of Design Thinking with the logical rigor of the Pyramid Principle and MECE structuring.

This reference produces a **Storyline Package in markdown** — the narrative backbone that Phase 2 (Slide Design) consumes.

## Phase Flow

```
DISCOVER → ARCHITECT → REFINE
```

**Discover** uses Design Thinking to understand the audience before writing a single word. **Architect** applies Pyramid Principle and MECE to build a logically airtight structure. **Refine** drafts the storyline and iterates with the user.

Always start with Discover. Never skip it.

<a id="discover"></a>
## DISCOVER (Design Thinking)

The goal: understand who is the audience, what they care about, and what tension your story must resolve.

### Step 1 — Empathy Interview

Ask the user these questions. Don't proceed until you have clear answers:

1. **Audience**: Who will be in the room? Roles, priorities, concerns?
2. **Current belief**: What does this audience currently believe about the topic?
3. **Desired shift**: What should they believe, feel, or decide after the presentation?
4. **Tension**: What is the obstacle? Why hasn't this been solved?
5. **Evidence**: What data, proof points, or stories do you have? What's missing?

If the user provides a brief or document, extract answers from the material and confirm before proceeding.

### Step 2 — Audience Empathy Map

Synthesize the interview into a map. This is your compass for every storyline decision.

```
AUDIENCE EMPATHY MAP
━━━━━━━━━━━━━━━━━━━
Audience:       [Who — roles, seniority, decision power]
They think:     [Current mental model about the topic]
They feel:      [Emotional state — confident? anxious? skeptical?]
They want:      [What outcome matters to them personally]
They fear:      [What risk or loss concerns them]
━━━━━━━━━━━━━━━━━━━
Desired shift:    [From current belief → to target belief]
Decision needed:  [Specific action/approval you need]
```

### Step 3 — Core Message

Distill one sentence that captures the entire presentation — the **governing thought**. It must pass three tests:

- **Relevance test**: Does it address what the audience cares about?
- **Tension test**: Does it acknowledge or resolve the conflict?
- **Action test**: Does it point toward a clear decision?

```
CORE MESSAGE
━━━━━━━━━━━━
✗ Bad:  "Digital transformation strategy"
✗ Bad:  "We should invest in cloud"
✓ Good: "Migrating core banking to cloud will reduce operational costs by 30%
         and position Itaú to launch new products 3x faster than competitors"
```

## ARCHITECT (Pyramid Principle + MECE)

Lead with the answer, group supporting arguments, provide evidence. Every level must be MECE.

### Step 4 — Build the Pyramid

Start from the core message, work top-down:

```
LEVEL 0 — GOVERNING THOUGHT (Core Message)
│
├── LEVEL 1 — KEY ARGUMENT A
│   ├── Level 2: Supporting point A1
│   ├── Level 2: Supporting point A2
│   └── Level 2: Supporting point A3
│
├── LEVEL 1 — KEY ARGUMENT B
│   ├── Level 2: Supporting point B1
│   ├── Level 2: Supporting point B2
│   └── Level 2: Supporting point B3
│
└── LEVEL 1 — KEY ARGUMENT C
    ├── Level 2: Supporting point C1
    ├── Level 2: Supporting point C2
    └── Level 2: Supporting point C3
```

Rules:
- **Level 1** (typically 3, occasionally 2 or 4) — MECE: each covers a distinct dimension, together they fully support the governing thought.
- **Level 2** — supports parent with evidence/data/logic, also MECE within group.
- **Grouping logic** at Level 1:
  - **Temporal**: Past → Present → Future
  - **Structural**: Market → Product → Operations
  - **Causal**: Problem → Root cause → Solution
  - **Stakeholder**: Customer → Company → Partner

### Step 5 — MECE Validation

Stress-test the pyramid before proceeding:

```
MECE CHECK
━━━━━━━━━━
□ Mutually Exclusive — Any overlap? → merge or redefine
□ Collectively Exhaustive — Anything missing? → add it
□ Same level of abstraction — Arguments parallel? → elevate or demote
□ Logical grouping — Pattern makes sense? → try different logic
```

### Step 6 — Narrative Arc

Transform the pyramid into a storyline with emotional flow. The pyramid gives logic; the arc gives momentum.

```
NARRATIVE ARC
━━━━━━━━━━━━━
    ╱╲          ← CLIMAX: Core insight or recommendation
   ╱  ╲
  ╱    ╲        ← RISING: Build evidence, create urgency
 ╱      ╲
╱        ╲      ← RESOLUTION: Path forward, next steps
━━━━━━━━━━╲━━━━
↑                ↑
HOOK             CALL TO ACTION
```

Map each pyramid element to the arc:

1. **Hook** (1 slide): Provocative data point, question, or contrast. Establishes tension.
2. **Context** (1-2 slides): Shared understanding. Validate current mental model before challenging it.
3. **Rising tension** (3-5 slides): Key arguments with evidence. Each builds urgency and shifts belief.
4. **Climax** (1-2 slides): Governing thought at full force. Data, insight, and narrative converge.
5. **Resolution** (2-3 slides): Path forward — roadmap, investment, milestones. Make action feel achievable.
6. **Call to action** (1 slide): Exactly what you need, by when.

<a id="refine"></a>
## REFINE (Rapid Prototyping)

### Step 7 — Storyline Draft

Output the full storyline in this format. Each line becomes one slide. Titles must be **action statements** (findings, not topics).

```
STORYLINE: [Presentation Title]
Audience: [From empathy map]
Core Message: [Governing thought]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[Arc]     #  | Action Title                                    | Exhibit Hint
----------|--------------------------------------------------|------------------
HOOK      |  1. [Provocative opening]                        | [visual suggestion]
CONTEXT   |  2. [Shared understanding]                       | [visual suggestion]
CONTEXT   |  3. [Challenge or opportunity framing]            | [visual suggestion]
RISING    |  4. [Key Argument A — with evidence]              | [visual suggestion]
RISING    |  5. [Deepening point for A]                       | [visual suggestion]
RISING    |  6. [Key Argument B — with evidence]              | [visual suggestion]
RISING    |  7. [Deepening point for B]                       | [visual suggestion]
RISING    |  8. [Key Argument C — with evidence]              | [visual suggestion]
CLIMAX    |  9. [Core message with full force]                | [visual suggestion]
CLIMAX    | 10. [Quantified impact]                           | [visual suggestion]
RESOLUT.  | 11. [Recommended approach / roadmap]              | [visual suggestion]
RESOLUT.  | 12. [Investment and resources]                    | [visual suggestion]
RESOLUT.  | 13. [Risk mitigation]                             | [visual suggestion]
CTA       | 14. [Decision needed + timeline]                  | [visual suggestion]
```

**Exhibit Hints** are suggestions for Phase 2 about what visual would support each slide. Examples: "bar chart Q1-Q4", "3 KPIs", "2x2 matrix", "timeline 3 phases", "customer quote", "before/after". These are guidance, not mandates.

### Step 8 — Self-Critique

Before presenting, run these checks:

- **Title-only test**: Read only titles in sequence. Does the story work? If not, rewrite.
- **Empathy test**: Does the storyline address what the audience fears, wants, believes?
- **"So what?" test**: Can you say in one sentence why the audience should care about each slide?
- **Skeptic test**: Would the most skeptical person's challenge break any argument?

Fix issues before showing the user.

### Step 9 — Iterate with the User

Present the storyline and invite feedback:
- "Does the opening create the right tension?"
- "Are the arguments the right ones, or should we reframe?"
- "Is the call to action realistic for this meeting?"

Revise and re-present until the storyline feels right. Then hand off to the orchestrator for Checkpoint 1 approval.

<a id="principles"></a>
## Principles

**Audience over author.** The storyline exists to move the audience, not showcase analysis. Every element justified by the empathy map.

**Logic earns trust, emotion drives action.** Pyramid = credibility. Narrative arc = persuasion. You need both.

**One message, relentlessly reinforced.** If someone asks "what was that about?", everyone should give the same one-sentence answer.

**Tension is your friend.** "Everything is fine and here's how to improve" falls flat. Show stakes.

**Less is more.** 12 slides with conviction beats 30 that hedge.
